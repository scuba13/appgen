# Design System Standard

## Obrigatorio

- Use Ant Design as the PoC design system.
- Replace Ant Design with the official corporate design system when the company profile declares one.
- Use existing component primitives before creating custom controls.
- Keep operational screens dense, predictable, and easy to scan.
- Provide consistent layout, navigation, tables, lists, forms, empty states, loading states, and error states.
- Meet accessibility requirements for keyboard use, labels, contrast, and focus.

## Recomendado

- Use shared design tokens when available.
- Use iconography from the active design system.
- Keep UI copy short, specific, and business-oriented.

## Proibido

- Ask the business user to choose component libraries.
- Mix unrelated component systems in the same app without an approved exception.
- Ship decorative UI that makes operational work harder to scan.
