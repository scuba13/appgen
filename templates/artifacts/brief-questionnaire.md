# Roteiro De Brief De Negocio

Use este roteiro para a primeira conversa com o usuario de negocio.

## Como Perguntar

- Faca perguntas curtas e em linguagem de negocio.
- Pergunte em blocos pequenos, no maximo 5 perguntas por rodada.
- Se o usuario ja explicou algo, nao pergunte de novo; confirme o entendimento.
- Quando faltar detalhe tecnico, registre para os agentes tecnicos. Nao repasse a decisao ao usuario.
- Se uma resposta estiver vaga, peca exemplo concreto do processo real.

## Rodada 1 - Objetivo

1. Qual problema esta app precisa resolver?
2. Quem sente esse problema hoje?
3. O que muda no trabalho das pessoas quando a app estiver pronta?
4. Existe algum processo atual que devemos respeitar?
5. Como voce vai saber que a app deu certo?

## Rodada 2 - Usuarios

1. Quem vai usar a app no dia a dia?
2. Existem perfis diferentes, como solicitante, aprovador, operador ou gestor?
3. O que cada perfil pode ver ou fazer?
4. Algum perfil precisa apenas consultar informacoes?
5. Existe alguma restricao de acesso por area, unidade, cliente ou cargo?

## Rodada 3 - Fluxos

1. Quais sao os fluxos obrigatorios da primeira versao?
2. Qual e o passo a passo do fluxo principal?
3. Em que momento o processo muda de status?
4. Quem precisa ser avisado quando algo muda?
5. Quais telas ou listas a pessoa espera encontrar?

## Rodada 4 - Regras

1. Quais regras nunca podem ser quebradas?
2. O que deve acontecer quando uma informacao obrigatoria faltar?
3. Existem limites, prazos, aprovacoes ou excecoes?
4. Que acoes devem ser bloqueadas para alguns perfis?
5. O que acontece quando alguem tenta fazer algo invalido?

## Rodada 5 - Dados

1. Quais informacoes precisam ser cadastradas ou consultadas?
2. Quais campos sao obrigatorios?
3. Existem dados sensiveis ou confidenciais?
4. Existe algum codigo, numero, documento ou identificador importante?
5. De onde esses dados vem hoje?

## Rodada 6 - Integracoes E Operacao

1. A app precisa receber ou enviar informacoes para algum sistema conhecido pelo negocio?
2. Existe e-mail, planilha, sistema interno ou processo manual que precisa continuar funcionando?
3. Existe prazo, compliance, contrato ou regulacao que afeta a app?
4. O que deve ficar fora da primeira versao?
5. Quem precisa aprovar o resultado final?

## Nao Perguntar Ao Usuario De Negocio

- Linguagem de programacao.
- Framework.
- Banco de dados.
- ORM.
- Arquitetura.
- Estrutura de pastas.
- Biblioteca de componentes.
- Ferramenta de teste.
- CI/CD.
- Cloud provider ou deploy, exceto quando for uma restricao contratual ja conhecida.

## Como Registrar Lacunas

Quando faltar resposta, registre em `Perguntas Abertas` com:

- pergunta objetiva;
- impacto no negocio;
- se bloqueia ou nao a primeira versao.
